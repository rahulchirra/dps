Cryptography Units

1 Classical Cryptography

Homophonic Substitution Cipher

Homophonic Substitution Cipher

A cipher where each plaintext character can map to multiple ciphertext characters. Example: 'A' could be replaced by 12, 34, or 56 randomly.

Plain: ATTACK → Cipher: 12 56 56 34 67 78

One Time Pad

One Time Pad

An encryption technique that cannot be cracked if used correctly. Requires a pre-shared key as long as the message, used only once.

Message: HELLO
Key: XMCKL
Cipher: EQNVZ (using XOR)

Polybius Cipher

Polybius Square Cipher

A 5x5 grid containing letters (I/J combined) where each letter is represented by its row and column numbers.

1 2 3 4 5
1 A B C D E
2 F G H I/J K
...
HELLO → 23 15 31 31 34

Playfair Cipher

Playfair Cipher

Digraph substitution cipher using a 5x5 key square. Kerckhoffs' principle states that security should depend only on the key.

Key: MONARCHY
HELLO → DM YG GN (after splitting and applying rules)

2 Polyalphabetic Ciphers

Gronsfeld Cipher

Gronsfeld Cipher

A variant of Vigenère cipher using numbers instead of letters for the key. Each digit shifts the corresponding plaintext letter.

Key: 123
HELLO → IGOMQ (H+1, E+2, L+3, L+1, O+2)

Beaufort Cipher

Beaufort Cipher

Similar to Vigenère but uses subtraction from the key character. Encryption and decryption use the same algorithm.

Key: KEY
HELLO → RIJVS (using Beaufort's formula)

Trithemius Cipher

Trithemius Cipher

An autokey cipher where the key is a fixed progression (like 0,1,2,...). A precursor to Vigenère cipher.

HELLO → HFLPP (H+0, E+1, L+2, L+3, O+4)

Vigenère vs Porta

Vigenère vs Porta Cipher

Vigenère uses modular addition while Porta uses a reciprocal table. Porta is more complex but both are polyalphabetic.

Vigenère: C = (P + K) mod 26
Porta: Uses 13x2 table with different alphabets

3 Modern Symmetric Cryptography

DES

Data Encryption Standard

A 56-bit symmetric key block cipher with 64-bit blocks. Uses Feistel network with 16 rounds. Now considered insecure.

Key expansion → Initial permutation → 16 rounds → Final permutation

RC5

RC5 Algorithm

A parameterizable block cipher with variable block size, key size, and number of rounds. Uses data-dependent rotations.

RC5-w/r/b where w=word size, r=rounds, b=key bytes
Common: RC5-32/12/16 (32-bit words, 12 rounds, 16-byte key)

Rijndael (AES)

Rijndael (AES)

Advanced Encryption Standard uses 128/192/256-bit keys. Operates on 4x4 byte matrices with SubBytes, ShiftRows, MixColumns, AddRoundKey.

10 rounds for 128-bit key
12 for 192-bit
14 for 256-bit

Dynamic Substitution

Dynamic Substitution

Substitution boxes (S-boxes) that change during encryption based on the key or plaintext, increasing security.

Example: S-boxes generated from key material using a PRNG

4 Asymmetric Cryptography

Elliptic Curve Cryptography

Elliptic Curve Cryptography

Uses algebraic structure of elliptic curves over finite fields for public-key cryptography. More efficient than RSA for equivalent security.

y² = x³ + ax + b
256-bit ECC ≈ 3072-bit RSA security

Diffie-Hellman

Diffie-Hellman Key Exchange

Method to securely exchange cryptographic keys over a public channel. Based on discrete logarithm problem.

Alice: g^a mod p
Bob: g^b mod p
Shared secret: g^(ab) mod p

PGP

Pretty Good Privacy

Data encryption/decryption program providing cryptographic privacy and authentication. Uses hybrid cryptosystem.

Combines symmetric (AES) and asymmetric (RSA) crypto
Digital signatures for authentication

RSA Cryptography

RSA Cryptography

Public-key cryptosystem based on the practical difficulty of factoring large integers. Used for secure data transmission.

Key generation: n=pq, φ(n), e, d
Encryption: c ≡ m^e mod n
Decryption: m ≡ c^d mod n

5 Data Hiding & Watermarking

Frequency Domain Watermarking

Robust Frequency Domain Watermarking

Embeds watermark in frequency domain (DCT, DWT) coefficients making it resistant to common attacks.

Process: Transform → Select coefficients → Modify → Inverse transform

LSB Encoding

Least Significant Bit Encoding

Hides data in the least significant bits of pixel values or audio samples. Simple but fragile to modifications.

Pixel: 11001010 → 11001011 (last bit changed)
Capacity: 1 bit per pixel for grayscale

Binary Image Data Hiding

Binary Image Data Hiding

Techniques for embedding data in binary images (like fax documents) using edge modification or pattern recognition.

Methods: Boundary modification, block permutation, pattern encoding

Wu-Lee Method

Wu-Lee Method

A binary image steganography method using block parity and flipping to embed data while preserving visual quality.

Process: Divide image into blocks → Calculate parity → Flip pixels if needed
